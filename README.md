# ReCall
学習内容を「思い出す」ための復習サポートアプリ

## サービス概要
学習内容を記述するだけでなくrecall = 思い出す　
「学習した内容を忘れないように、適切なタイミングで復習を促す」をコンセプトにしたアプリケーション

## アプリの特徴
* 学習内容を記録し、復習予定日を自動設定
* 間隔反復学習（SRS）による効果的な復習タイミング提案
* 「思い出せた/思い出せなかった」を記録し、学習の振り返りをサポート

## 想定されるユーザー層

* プログラミング学習者(RUNTEQの受講生など)
* 日々の学習内容を記録し、定期的に復習したい人
* 「以前学んだことを忘れてしまった」という経験がある人

## サービスの利用イメージ

### 学習記録
1. ユーザーが学習内容を記録 （例: 「Railsのルーティングについて学んだ」 ）
2. システムが自動で復習予定を設定 (1日後、3日後...と段階的に)

### 復習時
1. 復習予定日にメール通知が届く
2. 復習画面で学習内容を確認
3. 「思い出せた/思い出せなかった」を記録
4. 次の復習予定日が自動設定される

### 継続的な学習
- 復習を積み重ねることで、学習内容が長期記憶として定着
- 連続学習日数が記録され、モチベーション維持につながる

## サービスコンセプト

### このサービスが目指すもの
**「復習を習慣化し、学習内容の長期記憶への定着を支援する」**

本サービスは復習することで自体を目的とするのではなく、
**適切なタイミングでの復習を通じて、学習内容を確実に記憶に定着させること**に重点をおいています。

間隔反復練習(SRS)の仕組みを活用し、
「思い出す」プロセスを繰り返すことで、効率的な記憶の定着を実現します。

### 解決したい課題
* 多くの学習者が抱える課題:
「学習した内容が忘れてしまう」
* 復習のタイミングがわからない
* いつ復習したらいいのか判断できず、復習が後回しにしてしまう。
* 計画的な復習が続かない
  * 特に独学では、計画的な復習が難しい

エビングハウスの忘却曲線によれば
人は学習後24時間で約70%の内容を忘れると言われています。
適切なタイミングでの復習が、記憶定着の鍵となります。

### なぜこのサービスを作るのか
私自身、RUNTEQの学習を進める中で「以前学んだ内容を忘れている」という経験がありました。
エビングハウスの忘却曲線によれば、復習のタイミングが記憶定着に重要であることがわかっており、
間隔反復学習(SRS)を活用することで、効率的に知識を定着させられると考えました。

### 目指すサービス像
* 単なる学習記録アプリではなく、「想起学習」を促すことで記憶の定着を支援するアプリを目指します。
また今後の機能でユーザーが自然と復習習慣を身につけるように、リマインド機能など色々な機能を追加します。
### 差別化ポイント
* 想起学習に特化: 単なる記録アプリではなく、「思い出す」プロセスを重視しています。
* 復習タイミングの自動提案: 間隔反復学習(SRS)アルゴリズムによる最適な復習案を提案。
* シンプルで続けやすい機能: 複雑な機能を避け、学習の継続に集中できるような設計。

## 実装を予定している機能
### MVPリリース

#### 1. ユーザー管理（シンプルなログイン）
* メールアドレスとパスワードによる認証
* ユーザー登録・ログイン・ログアウト

#### 2. 学習記録機能
* 学習日（必須）
* 学習内容（テキストエリア）
* カテゴリ（選択式: プログラミング、語学、資格など）
* 復習予定日 (自動設定: 1日後 → 3日後 → 7日後 → 14日後 → 30日後 )

#### 3. 復習記録機能
* 復習時に「思い出した/思い出せなかった」の２択を記録
* 判断基準はユーザーの主観に委ねる
  * 「思い出せた」 = 学習内容の要点を説明できる程度
  * 「思い出せなかった」 = 内容を見ても思い出せない、または曖昧
* 「思い出せなかった」場合でも復習日はリセットしない
  * 理由：復習リストが溜まりすぎて学習が憂鬱になることを防ぐため

#### 4. メール通知機能
* リマインドをメールで通知  (MVP開発時のみ)
* Action MailerとActive Jobを使用

### 本リリース (MVP後に追加予定)
#### 5. カレンダー表示
* 学習日と復習予定日を色分け表示(gem'simple_calendar'など)で実装

#### 6. 復習専用画面
* 「内容を確認する」ボタンで非同期表示(Turbo,Stimulus)
* 苦手な内容は別途「苦手リスト」として管理
* フラッシュカード形式での復習

#### 7. 苦手分野の克服サポート
* 「思い出せなかった」記録が多い内容を「苦手リスト」として表示
* 苦手な内容の復習間隔を短縮するオプション
* ユーザーが復習間隔を調整できる機能

#### 8. 学習の継続をサポートする機能
* 連続学習日数の記録
* 学習ログのタイムライン表示

#### 9. 通知のパーソナライズの追加
* LINE Messaging APIによる通知機能によるリマインダー (復習タイミングの通知)
